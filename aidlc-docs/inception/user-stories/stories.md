# 사용자 스토리 (User Stories)

스토리 분류: Feature 기반 | 수용 기준: Given-When-Then (BDD) | 세분화: 논리적 단위 | 우선순위: MoSCoW

---

## 고객용 스토리 (Customer Stories)

### US-C01: 테이블 자동 로그인 및 세션 관리
**As a** 고객,
**I want** 테이블 태블릿에서 별도 로그인 없이 자동으로 인증되기를,
**So that** 즉시 메뉴를 탐색하고 주문할 수 있다.

**Priority**: Must Have
**Persona**: 고객 (김민수)
**Requirement**: FR-C01

**Acceptance Criteria**:

1. Given 관리자가 테이블 초기 설정(매장 ID, 테이블 번호, 비밀번호)을 완료한 상태일 때,
   When 태블릿 브라우저에서 서비스에 접속하면,
   Then 저장된 정보로 자동 로그인되어 메뉴 화면이 표시된다.

2. Given 자동 로그인 정보가 로컬에 저장되어 있을 때,
   When 페이지를 새로고침하면,
   Then 로그인 상태가 유지되고 메뉴 화면이 다시 표시된다.

3. Given 저장된 로그인 정보가 유효하지 않을 때,
   When 자동 로그인을 시도하면,
   Then 에러 메시지가 표시되고 초기 설정 화면으로 이동한다.

---

### US-C02: 메뉴 조회 및 탐색
**As a** 고객,
**I want** 카테고리별로 메뉴를 탐색하고 상세 정보를 확인하기를,
**So that** 원하는 메뉴를 쉽게 찾아 선택할 수 있다.

**Priority**: Must Have
**Persona**: 고객 (김민수)
**Requirement**: FR-C02

**Acceptance Criteria**:

1. Given 메뉴 화면에 접속한 상태일 때,
   When 화면이 로드되면,
   Then 카테고리별로 분류된 메뉴 목록이 카드 형태로 표시된다.

2. Given 메뉴 목록이 표시된 상태일 때,
   When 특정 카테고리를 선택하면,
   Then 해당 카테고리의 메뉴만 필터링되어 표시된다.

3. Given 메뉴 카드가 표시된 상태일 때,
   When 메뉴 카드를 확인하면,
   Then 메뉴명, 가격, 설명, 이미지가 모두 표시된다.

4. Given 메뉴가 여러 카테고리에 걸쳐 있을 때,
   When 카테고리 간 이동을 하면,
   Then 빠르게 해당 카테고리로 스크롤/전환된다.

5. Given 메뉴 데이터 로딩에 실패했을 때,
   When 화면이 로드되면,
   Then 에러 메시지가 표시되고 재시도 옵션이 제공된다.

---

### US-C03: 장바구니 관리
**As a** 고객,
**I want** 주문 전에 선택한 메뉴를 장바구니에서 확인하고 수정하기를,
**So that** 최종 주문 전에 내역과 금액을 정확히 확인할 수 있다.

**Priority**: Must Have
**Persona**: 고객 (김민수)
**Requirement**: FR-C03

**Acceptance Criteria**:

1. Given 메뉴 화면에서 메뉴를 선택한 상태일 때,
   When 장바구니에 추가 버튼을 누르면,
   Then 해당 메뉴가 장바구니에 추가되고 총 금액이 업데이트된다.

2. Given 장바구니에 메뉴가 있는 상태일 때,
   When 수량 증가/감소 버튼을 누르면,
   Then 수량이 변경되고 총 금액이 실시간으로 재계산된다.

3. Given 장바구니에 메뉴가 있는 상태일 때,
   When 특정 메뉴를 삭제하면,
   Then 해당 메뉴가 장바구니에서 제거되고 총 금액이 재계산된다.

4. Given 장바구니에 메뉴가 있는 상태일 때,
   When 장바구니 비우기를 실행하면,
   Then 모든 메뉴가 제거되고 총 금액이 0원이 된다.

5. Given 장바구니에 메뉴가 있는 상태일 때,
   When 페이지를 새로고침하면,
   Then 장바구니 내용이 로컬 저장소에서 복원되어 유지된다.

6. Given 장바구니가 비어있는 상태일 때,
   When 장바구니를 확인하면,
   Then 빈 장바구니 안내 메시지가 표시된다.

---

### US-C04: 주문 생성
**As a** 고객,
**I want** 장바구니의 메뉴를 주문으로 확정하기를,
**So that** 매장에서 음식을 준비받을 수 있다.

**Priority**: Must Have
**Persona**: 고객 (김민수)
**Requirement**: FR-C04

**Acceptance Criteria**:

1. Given 장바구니에 1개 이상의 메뉴가 있는 상태일 때,
   When 주문 확정 버튼을 누르면,
   Then 주문 내역 최종 확인 화면이 표시된다.

2. Given 주문 확인 화면에서 내역을 확인한 상태일 때,
   When 최종 주문 확정을 하면,
   Then 주문이 서버에 전송되고 주문 번호가 표시된다.

3. Given 주문이 성공적으로 생성된 상태일 때,
   When 주문 성공 화면이 표시되면,
   Then 장바구니가 자동으로 비워지고 5초 후 메뉴 화면으로 자동 리다이렉트된다.

4. Given 주문 전송 중 서버 오류가 발생했을 때,
   When 주문 확정을 시도하면,
   Then 에러 메시지가 표시되고 장바구니 내용은 유지된다.

5. Given 장바구니가 비어있는 상태일 때,
   When 주문 확정을 시도하면,
   Then 주문 버튼이 비활성화되어 있거나 안내 메시지가 표시된다.

---

### US-C05: 주문 내역 조회
**As a** 고객,
**I want** 현재 테이블 세션의 주문 내역과 상태를 확인하기를,
**So that** 주문 진행 상황을 실시간으로 파악할 수 있다.

**Priority**: Must Have
**Persona**: 고객 (김민수)
**Requirement**: FR-C05

**Acceptance Criteria**:

1. Given 현재 세션에서 주문이 존재하는 상태일 때,
   When 주문 내역 화면에 접속하면,
   Then 주문 시간 순으로 정렬된 주문 목록이 표시된다.

2. Given 주문 내역이 표시된 상태일 때,
   When 각 주문을 확인하면,
   Then 주문 번호, 주문 시각, 메뉴 및 수량, 금액, 상태(대기중/준비중/완료)가 표시된다.

3. Given 관리자가 주문 상태를 변경한 상태일 때,
   When 고객이 주문 내역 화면을 보고 있으면,
   Then SSE를 통해 주문 상태가 실시간으로 업데이트된다.

4. Given 이전 테이블 세션의 주문이 있는 상태일 때,
   When 주문 내역 화면에 접속하면,
   Then 현재 세션의 주문만 표시되고 이전 세션 주문은 제외된다.

5. Given 현재 세션에 주문이 없는 상태일 때,
   When 주문 내역 화면에 접속하면,
   Then 주문 내역이 없다는 안내 메시지가 표시된다.

## 관리자용 스토리 (Admin Stories)

### US-A01: 매장 인증
**As a** 매장 관리자,
**I want** 매장 관리 시스템에 안전하게 로그인하기를,
**So that** 매장 운영을 관리할 수 있다.

**Priority**: Must Have
**Persona**: 관리자 (박서연)
**Requirement**: FR-A01

**Acceptance Criteria**:

1. Given 관리자 로그인 화면에 접속한 상태일 때,
   When 매장 식별자, 사용자명, 비밀번호를 입력하고 로그인하면,
   Then JWT 토큰이 발급되고 관리자 대시보드로 이동한다.

2. Given 로그인에 성공한 상태일 때,
   When 브라우저를 새로고침하면,
   Then 세션이 유지되어 재로그인 없이 대시보드가 표시된다.

3. Given 로그인 후 16시간이 경과한 상태일 때,
   When 관리자가 작업을 시도하면,
   Then 자동 로그아웃되고 로그인 화면으로 이동한다.

4. Given 잘못된 인증 정보를 입력한 상태일 때,
   When 로그인을 시도하면,
   Then 에러 메시지가 표시되고 로그인 시도 횟수가 제한된다.

---

### US-A02: 실시간 주문 모니터링
**As a** 매장 관리자,
**I want** 들어오는 주문을 실시간으로 모니터링하고 상태를 관리하기를,
**So that** 주문을 빠르게 처리하고 매장 운영 현황을 파악할 수 있다.

**Priority**: Must Have
**Persona**: 관리자 (박서연)
**Requirement**: FR-A02

**Acceptance Criteria**:

1. Given 관리자 대시보드에 접속한 상태일 때,
   When 화면이 로드되면,
   Then 테이블별 카드 형태의 그리드 레이아웃으로 주문 현황이 표시된다.

2. Given 대시보드가 표시된 상태일 때,
   When 고객이 새로운 주문을 생성하면,
   Then SSE를 통해 2초 이내에 해당 테이블 카드에 신규 주문이 표시되고 시각적으로 강조된다.

3. Given 테이블 카드가 표시된 상태일 때,
   When 카드를 확인하면,
   Then 테이블 번호, 총 주문액, 최신 주문 미리보기가 표시된다.

4. Given 테이블 카드를 클릭한 상태일 때,
   When 상세 보기가 열리면,
   Then 해당 테이블의 전체 주문 목록(주문 번호, 시각, 메뉴/수량, 금액)이 표시된다.

5. Given 주문 상세를 확인하는 상태일 때,
   When 주문 상태 변경 버튼(대기중→준비중→완료)을 누르면,
   Then 주문 상태가 즉시 변경되고 고객 화면에도 실시간 반영된다.

6. Given 대시보드에 여러 테이블이 표시된 상태일 때,
   When 특정 테이블로 필터링하면,
   Then 선택한 테이블의 주문만 표시된다.

### US-A03: 테이블 관리
**As a** 매장 관리자,
**I want** 테이블 설정, 주문 삭제, 이용 완료 처리를 하기를,
**So that** 테이블 회전율을 효율적으로 관리할 수 있다.

**Priority**: Must Have
**Persona**: 관리자 (박서연)
**Requirement**: FR-A03

**Acceptance Criteria**:

1. Given 테이블 관리 화면에 접속한 상태일 때,
   When 새 테이블 초기 설정(테이블 번호, 비밀번호)을 입력하면,
   Then 16시간 세션이 생성되고 해당 테이블의 자동 로그인이 활성화된다.

2. Given 테이블에 주문이 있는 상태일 때,
   When 특정 주문의 삭제 버튼을 누르면,
   Then 확인 팝업이 표시되고, 확인 시 주문이 삭제되며 총 주문액이 재계산된다.

3. Given 주문 삭제가 완료된 상태일 때,
   When 결과를 확인하면,
   Then 성공/실패 피드백 메시지가 표시된다.

4. Given 테이블에 활성 세션이 있는 상태일 때,
   When 이용 완료 버튼을 누르면,
   Then 확인 팝업이 표시되고, 확인 시 해당 세션의 주문 내역이 과거 이력으로 이동한다.

5. Given 테이블 이용 완료 처리가 된 상태일 때,
   When 해당 테이블을 확인하면,
   Then 현재 주문 목록과 총 주문액이 0으로 리셋되어 새 고객이 이전 내역 없이 시작할 수 있다.

6. Given 테이블 관리 화면에서 과거 내역 버튼을 누른 상태일 때,
   When 과거 주문 내역 화면이 열리면,
   Then 테이블별 과거 주문 목록이 시간 역순으로 표시되고 날짜 필터링이 가능하다.

7. Given 과거 주문 내역을 확인하는 상태일 때,
   When 각 주문을 확인하면,
   Then 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각이 표시된다.

---

### US-A04: 메뉴 관리
**As a** 매장 관리자,
**I want** 메뉴를 등록, 수정, 삭제하고 노출 순서를 조정하기를,
**So that** 매장 메뉴를 최신 상태로 유지할 수 있다.

**Priority**: Must Have
**Persona**: 관리자 (박서연)
**Requirement**: FR-A04

**Acceptance Criteria**:

1. Given 메뉴 관리 화면에 접속한 상태일 때,
   When 화면이 로드되면,
   Then 카테고리별로 분류된 메뉴 목록이 표시된다.

2. Given 메뉴 등록 화면에서 필수 정보(메뉴명, 가격, 카테고리)를 입력한 상태일 때,
   When 메뉴 등록 버튼을 누르면,
   Then 새 메뉴가 등록되고 메뉴 목록에 반영된다.

3. Given 메뉴 등록 시 이미지를 첨부한 상태일 때,
   When 등록을 완료하면,
   Then 이미지가 서버에 업로드되고 메뉴에 이미지가 표시된다.

4. Given 기존 메뉴를 수정하는 상태일 때,
   When 메뉴 정보(메뉴명, 가격, 설명, 카테고리, 이미지)를 변경하고 저장하면,
   Then 변경 사항이 즉시 반영되고 고객 화면에도 업데이트된다.

5. Given 메뉴를 삭제하려는 상태일 때,
   When 삭제 버튼을 누르면,
   Then 확인 팝업이 표시되고, 확인 시 메뉴가 삭제된다.

6. Given 메뉴 목록이 표시된 상태일 때,
   When 메뉴 노출 순서를 드래그 앤 드롭으로 변경하면,
   Then 변경된 순서가 저장되고 고객 화면에 반영된다.

7. Given 메뉴 등록 시 필수 필드가 비어있거나 가격이 유효 범위를 벗어난 상태일 때,
   When 등록/수정을 시도하면,
   Then 유효성 검증 에러 메시지가 표시되고 저장이 차단된다.
