# 사용자 페르소나 (User Personas)

---

## Persona 1: 고객 (Customer) — "테이블 이용자"

| 항목 | 내용 |
|---|---|
| 이름 | 김민수 (대표 페르소나) |
| 역할 | 매장 방문 고객 |
| 연령대 | 20~50대 (폭넓은 연령층) |
| 기술 수준 | 초~중급 (스마트폰/태블릿 기본 사용 가능) |
| 사용 환경 | 매장 내 테이블에 설치된 태블릿 (브라우저 기반) |

### 목표 (Goals)
- 대기 시간 없이 빠르게 메뉴를 주문하고 싶다
- 메뉴 정보(이미지, 가격, 설명)를 한눈에 확인하고 싶다
- 주문한 내역과 현재 상태를 실시간으로 확인하고 싶다
- 추가 주문을 간편하게 하고 싶다

### 동기 (Motivations)
- 직원을 부르지 않고 편하게 주문하는 경험
- 주문 실수를 줄이고 싶음 (장바구니에서 확인 후 주문)
- 현재 총 주문 금액을 미리 파악하고 싶음

### 불편사항 (Pain Points)
- 복잡한 인터페이스나 로그인 절차
- 주문 후 상태를 알 수 없는 불안감
- 메뉴 정보가 부족하거나 이미지가 없는 경우
- 페이지가 느리거나 반응이 없는 경우

### 주요 사용 시나리오
1. 테이블에 앉아 태블릿으로 메뉴를 탐색한다
2. 원하는 메뉴를 장바구니에 담고 수량을 조절한다
3. 주문을 확정하고 주문 번호를 확인한다
4. 식사 중 추가 주문을 한다
5. 주문 내역에서 현재 상태를 확인한다

## Persona 2: 매장 관리자 (Admin) — "매장 운영자"

| 항목 | 내용 |
|---|---|
| 이름 | 박서연 (대표 페르소나) |
| 역할 | 매장 사장/매니저 |
| 연령대 | 30~50대 |
| 기술 수준 | 중급 (PC/태블릿 업무 사용 가능) |
| 사용 환경 | 매장 카운터 PC 또는 태블릿 (브라우저 기반) |

### 목표 (Goals)
- 들어오는 주문을 실시간으로 빠르게 확인하고 처리하고 싶다
- 테이블별 주문 현황을 한눈에 파악하고 싶다
- 메뉴를 쉽게 등록/수정/삭제하고 싶다
- 테이블 회전율을 효율적으로 관리하고 싶다

### 동기 (Motivations)
- 인건비 절감 및 운영 효율성 향상
- 주문 누락이나 실수 방지
- 매장 운영 상황을 실시간으로 모니터링
- 고객 만족도 향상

### 불편사항 (Pain Points)
- 주문이 들어왔는데 바로 확인하지 못하는 상황
- 테이블별 주문 현황 파악이 어려운 경우
- 메뉴 변경 시 복잡한 절차
- 이전 고객의 주문 내역이 새 고객에게 보이는 문제

### 주요 사용 시나리오
1. 매장 오픈 시 관리자 로그인
2. 대시보드에서 실시간 주문 모니터링
3. 신규 주문 확인 후 상태 변경 (대기중 → 준비중 → 완료)
4. 잘못된 주문 삭제 처리
5. 고객 퇴장 시 테이블 이용 완료 처리
6. 메뉴 등록/수정/삭제
7. 과거 주문 내역 조회
