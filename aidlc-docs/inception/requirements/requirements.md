# 테이블오더 서비스 — 요구사항 정의서

## Intent Analysis

- **사용자 요청**: 디지털 테이블오더 플랫폼 구축 (고객 주문 + 매장 관리)
- **요청 유형**: New Project (신규 프로젝트)
- **범위 추정**: Multiple Components (다중 컴포넌트 — 고객 UI, 관리자 UI, 백엔드 API, 데이터베이스)
- **복잡도 추정**: Moderate~Complex (실시간 통신, 다중 매장, 세션 관리 등)

---

## 1. 기술 스택 (Tech Stack)

| 계층 | 기술 |
|---|---|
| 프론트엔드 | React |
| 백엔드 | Node.js / Express |
| 데이터베이스 | PostgreSQL |
| 실시간 통신 | Server-Sent Events (SSE) |
| 인증 | JWT 토큰 기반 |
| 배포 환경 | AWS 클라우드 (EC2, RDS, S3 등) |

## 2. 기능 요구사항 (Functional Requirements)

### 2.1 고객용 기능 (Customer Features)

#### FR-C01: 테이블 태블릿 자동 로그인 및 세션 관리
- 관리자가 1회 초기 설정 (매장 식별자, 테이블 번호, 테이블 비밀번호)
- 로그인 정보 로컬 저장 후 자동 로그인
- 테이블 세션: 첫 주문 시작 ~ 이용 완료 처리까지

#### FR-C02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본(홈) 화면
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지
- 카테고리 간 빠른 이동
- 카드 형태 레이아웃, 터치 친화적 버튼 (최소 44x44px)

#### FR-C03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절 (증가/감소)
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 로컬 저장 (페이지 새로고침 시 유지)
- 서버 전송은 주문 확정 시에만 수행

#### FR-C04: 주문 생성
- 주문 내역 최종 확인 후 주문 확정
- 주문 성공 시: 주문 번호 표시 → 장바구니 자동 비우기 → 5초 후 메뉴 화면 자동 리다이렉트
- 주문 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID

#### FR-C05: 주문 내역 조회
- 현재 테이블 세션의 주문만 표시 (이전 세션 제외)
- 주문 시간 순 정렬
- 주문별 상세: 주문 번호, 주문 시각, 메뉴 및 수량, 금액, 상태(대기중/준비중/완료)
- SSE 기반 주문 상태 실시간 업데이트 (MVP 포함)
- 페이지네이션 또는 무한 스크롤

### 2.2 관리자용 기능 (Admin Features)

#### FR-A01: 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 입력
- JWT 토큰 기반 인증, 16시간 세션 유지
- 브라우저 새로고침 시 세션 유지
- 16시간 후 자동 로그아웃
- 비밀번호 bcrypt 해싱
- 관리자 계정은 시스템 초기 데이터(seed data)로 생성

#### FR-A02: 실시간 주문 모니터링
- SSE 기반 주문 목록 실시간 업데이트 (2초 이내 표시)
- 그리드/대시보드 레이아웃: 테이블별 카드 형태
- 각 테이블 카드: 총 주문액, 최신 주문 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중 → 준비중 → 완료)
- 신규 주문 시각적 강조 (색상 변경, 애니메이션)
- 테이블별 필터링

#### FR-A03: 테이블 관리
- **초기 설정**: 테이블 번호 및 비밀번호 설정, 16시간 세션 생성
- **주문 삭제**: 확인 팝업 → 즉시 삭제 → 총 주문액 재계산 → 피드백
- **테이블 세션 종료(이용 완료)**: 확인 팝업 → 주문 내역 과거 이력 이동 → 현재 주문/총액 리셋 → 피드백
- **과거 주문 내역 조회**: 테이블별 과거 주문 목록 (시간 역순), 날짜 필터링, 닫기 버튼

#### FR-A04: 메뉴 관리 (MVP 포함)
- 메뉴 조회 (카테고리별)
- 메뉴 등록: 메뉴명, 가격, 설명, 카테고리, 이미지 (서버 업로드)
- 메뉴 수정
- 메뉴 삭제
- 메뉴 노출 순서 조정
- 필수 필드 검증, 가격 범위 검증

## 3. 비기능 요구사항 (Non-Functional Requirements)

### NFR-01: 성능
- 동시 접속자 50명 이하 지원
- SSE 기반 실시간 통신 — 주문 표시 2초 이내
- 페이지 로딩 시간 3초 이내

### NFR-02: 확장성
- 다중 매장 지원 (각 매장별 독립 데이터)
- 매장당 테이블 수 제한 없음 (유동적)

### NFR-03: 인증 및 보안
- JWT 토큰 기반 인증 (관리자 16시간, 테이블 16시간)
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한
- (보안 확장 규칙은 적용하지 않음 — MVP/프로토타입 수준)

### NFR-04: 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- 직관적인 카드 기반 레이아웃
- 명확한 시각적 계층 구조
- 반응형 디자인

### NFR-05: 데이터 관리
- 주문 이력 데이터베이스 영구 저장 (OrderHistory)
- 세션 ID 기반 주문 그룹화
- 장바구니 클라이언트 측 로컬 저장

### NFR-06: 이미지 관리
- 서버에 이미지 파일 업로드 기능
- AWS S3 저장 (배포 환경 기준)

## 4. 제외 범위 (Out of Scope)

요구사항 제약 문서(`constraints.md`) 기준:
- 결제 처리 (PG사 연동, 영수증, 환불, 포인트/쿠폰)
- 복잡한 인증 (OAuth, SNS 로그인, 2FA)
- 이미지 리사이징/최적화, CMS, 광고
- 알림 시스템 (푸시, SMS, 이메일, 소리/진동)
- 주방 기능 (주방 전달, 재고 관리)
- 고급 기능 (데이터 분석, 매출 리포트, 직원 관리, 예약, 리뷰, 다국어)
- 외부 연동 (배달 플랫폼, POS, 소셜 미디어, 지도/번역 API)
